<p align="center"><sub><em>
I got tired of watching AI agents make the same mistake twice with no way to know why.<br />
So I built a diff engine. Then I needed to know when the facts it relied on had gone stale. So I built a staleness monitor.<br />
Twelve libraries later, I had accidentally named the substrate layer of the agentic era.<br />
Not frameworks. Not wrappers. The missing primitives — the things every serious agent stack eventually needs but nobody had bothered to build cleanly.<br />
One sharp tool at a time.
</em></sub></p>

**[salluru.dev →](https://salluru.dev)**

## Now

**Building 12 open-source Python libraries** for the full agentic AI infrastructure stack — agent observability, state management, execution attestation, computer-use primitives, multi-agent coordination, and game AI. All MIT-licensed, all on PyPI, all with CLI + REST API + MCP server interfaces.

Each one is a focused tool that solves exactly one problem and solves it well. No monorepo. No framework lock-in. Drop any of them into any agent stack in five minutes.
